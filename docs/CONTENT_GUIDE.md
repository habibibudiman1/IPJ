# Content Management Guide

This guide helps content managers update website content through Sanity CMS.

## Accessing Sanity Studio

### Production URL
`https://your-project.sanity.studio`

### Local Development
```bash
cd sanity
npm run dev
```
Then open `http://localhost:3333`

## Login Credentials

Use your Sanity account credentials to log in. Contact the administrator if you don't have access.

## Content Sections

### 1. Company Profile

**Location:** Sanity Studio > Company Profile

**What to edit:**
- Company description and overview
- Core supply focus list
- Supporting ingredients list
- Quality statement

**Best practices:**
- Keep descriptions clear and professional
- Use bullet points for lists
- Emphasize quality and reliability

### 2. Vision & Mission

**Location:** Sanity Studio > Vision & Mission

**What to edit:**
- Vision statement
- Mission statements (multiple items)

**Best practices:**
- Keep vision inspiring but realistic
- Make missions specific and actionable
- Use active voice

### 3. Core Values

**Location:** Sanity Studio > Core Values

**What to edit:**
- Value name
- Value description
- Icon name (from Lucide icons)

**Available icons:**
- Award (Quality First)
- Shield (Integrity)
- Lightbulb (Innovation)
- Users (Customer Focus)
- Leaf (Sustainability)
- Handshake (Team Work)

**Best practices:**
- Keep descriptions concise (2-3 sentences)
- Focus on real practices, not aspirations
- Maintain consistent tone

### 4. Products

**Location:** Sanity Studio > Products

**What to edit:**
- Product name
- Description
- Product image
- Key features (list)
- Specifications

**Image guidelines:**
- Format: JPG or PNG
- Size: 1200x800px minimum
- Quality: High resolution
- Background: Clean, professional

**Best practices:**
- Use clear, benefit-focused language
- List features as bullet points
- Include technical specifications
- Update images regularly

### 5. Product Applications

**Location:** Sanity Studio > Product Applications

**What to edit:**
- Product name (must match Products)
- Applications list

**Best practices:**
- List real-world use cases
- Be specific (e.g., "Bakery fillings" not just "Bakery")
- Order by importance/popularity

### 6. Our Advantages

**Location:** Sanity Studio > Our Advantages

**What to edit:**
- Advantage title
- Description
- Icon name

**Best practices:**
- Focus on unique selling points
- Use concrete examples
- Keep descriptions brief but meaningful

### 7. Client Segments

**Location:** Sanity Studio > Client Segments

**What to edit:**
- Segment name
- Icon name

**Best practices:**
- List actual target markets
- Use industry-standard terminology
- Keep list focused (5-7 segments)

### 8. Contact Information

**Location:** Sanity Studio > Contact Information

**What to edit:**
- Physical address
- Phone number
- Email address
- Business hours

**Important:**
- Double-check accuracy
- Include country code for phone
- Use professional email address
- Update hours for holidays

### 9. Gallery

**Location:** Sanity Studio > Gallery

**What to edit:**
- Image title
- Category (Products, Facilities, Team, Certifications)
- Image file
- Description

**Image guidelines:**
- **Products:** Clear product shots, white/neutral background
- **Facilities:** Clean, well-lit facility photos
- **Team:** Professional team photos
- **Certifications:** High-quality scans of certificates

**Best practices:**
- Use high-resolution images
- Optimize file size (compress before upload)
- Add descriptive titles
- Include relevant keywords in descriptions

## Publishing Changes

### How Changes Work

1. **Edit Content:** Make your changes in Sanity Studio
2. **Preview:** Changes appear immediately in preview mode
3. **Publish:** Click "Publish" to make changes live
4. **Website Update:** Website updates automatically (may take a few seconds)

### Before Publishing Checklist

- [ ] Check for typos and grammar
- [ ] Verify all links work
- [ ] Confirm images look good
- [ ] Test on mobile preview
- [ ] Review changes in preview mode

## Best Practices

### Writing Style

**Do:**
- Use active voice
- Be specific and concrete
- Focus on benefits to customers
- Keep sentences short and clear
- Use professional language

**Don't:**
- Use jargon without explanation
- Make unverifiable claims
- Write overly long paragraphs
- Use informal language
- Include personal opinions

### Images

**Do:**
- Use high-quality professional photos
- Optimize images before uploading
- Use consistent image style
- Include alt text for accessibility
- Show real products/facilities

**Don't:**
- Use stock photos excessively
- Upload very large files (>5MB)
- Use blurry or pixelated images
- Include watermarked images
- Use copyrighted images without permission

### SEO Tips

- Use relevant keywords naturally
- Write descriptive titles
- Include location information
- Update content regularly
- Use proper headings structure

## Common Tasks

### Adding a New Product

1. Go to Products
2. Click "Create new Product"
3. Fill in all fields:
   - Name
   - Slug (auto-generated from name)
   - Description
   - Upload image
   - Add features
   - Add specifications
4. Click "Publish"
5. Go to Product Applications
6. Add applications for the new product
7. Click "Publish"

### Updating Contact Information

1. Go to Contact Information
2. Update the relevant field
3. Verify accuracy
4. Click "Publish"
5. Check website to confirm changes

### Changing Company Description

1. Go to Company Profile
2. Edit the description text
3. Preview changes
4. Click "Publish"

### Adding Gallery Images

1. Go to Gallery
2. Click "Create new Gallery"
3. Add title
4. Select category
5. Upload image
6. Add description
7. Click "Publish"

## Troubleshooting

### Changes Not Appearing

**Solution:**
1. Wait 30-60 seconds for CDN cache
2. Refresh browser (Ctrl+F5 or Cmd+Shift+R)
3. Clear browser cache
4. Check if changes were published (not just saved as draft)

### Image Upload Fails

**Solution:**
1. Check file size (should be under 10MB)
2. Convert to JPG or PNG format
3. Try compressing image
4. Check internet connection

### Can't Log In

**Solution:**
1. Verify you're using correct Sanity credentials
2. Reset password if needed
3. Contact administrator for access

### Content Looks Different on Website

**Solution:**
1. Check if using correct formatting
2. Preview before publishing
3. Test on different devices
4. Contact developer if issue persists
